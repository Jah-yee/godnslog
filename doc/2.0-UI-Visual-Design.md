### 9.2 暗黑模式主题切换实现

```
主题切换按钮位置: 顶部栏右侧用户菜单旁

[🌙 Light/Dark/System ▼]

切换方式:
1. 系统跟随 (默认) - 使用 prefers-color-scheme 媒体查询
2. 手动 Light - 强制浅色主题
3. 手动 Dark - 强制深色主题

实现细节:
- 使用 CSS Variables 定义主题色
- 在 :root 中定义 Light 模式变量
- 在 [data-theme="dark"] 中覆盖 Dark 模式变量
- 使用 localStorage 持久化用户选择
- 添加平滑过渡: transition: background-color 0.3s, color 0.3s
```

## 十、关键交互动效设计

### 10.1 页面切换动画

淡入淡出 (Fade In/Out):
- 持续时间: 200ms
- 缓动函数: ease-out
- 应用场景: 页面路由切换

实现:
.opacity-0 { opacity: 0; }
.opacity-100 { opacity: 1; }
.transition-opacity { transition: opacity 200ms ease-out; }

### 10.2 数据加载动画

骨架屏 (Skeleton Loading):
- 持续时间: 持续直到数据加载完成
- 应用场景: 表格、卡片、图表数据加载
- 颜色: gray-200 (Light) / gray-700 (Dark)
- 动画: 1.5s 闪烁循环
- 实现: 使用shadcn/ui的Skeleton组件

### 10.3 按钮点击反馈

按钮点击动画:
- 持续时间: 100ms
- 缓动函数: ease-in-out
- 效果: scale(0.95) → scale(1)
- 应用场景: 所有交互按钮

### 10.4 复制成功提示

Toast通知:
- 持续时间: 3000ms
- 位置: 右下角
- 动画: slide-in from right, fade-out
- 内容: "复制成功" / "已复制到剪贴板"

### 10.5 实时数据更新

WebSocket/SSE推送:
- 更新频率: 实时推送
- 动画: 新增项淡入, 更新项闪烁
- 应用场景: Interaction Timeline实时命中流
- 声音通知: 可选, 默认关闭

## 十三、边界状态设计

### 13.1 错误状态 (Error State)

图标: ❌ (red-500)
主标题: 加载失败 (red-600)
副标题: 简短错误说明 (gray-500)
操作: 重试按钮 (Primary色), 返回首页 (Secondary色)

### 13.2 加载状态 (Loading State)

使用骨架屏 (Skeleton Loading):
- 持续时间: 持续直到数据加载完成
- 颜色: gray-200 (Light) / gray-700 (Dark)
- 动画: 1.5s 闪烁循环
- 实现: 使用shadcn/ui的Skeleton组件

### 13.3 网络异常 (Network Error)

图标: 📡 (orange-500)
主标题: 网络连接失败 (orange-600)
副标题: 请检查您的网络连接 (gray-500)
操作: 重新连接 (Primary色),线模式 (Secondary色)

### 13.4 权限错误 (Permission Error)

图标: 🔒 (red-500)
主标题: 访问被拒绝 (red-600)
副标题: 您没有权限访问此资源 (gray-500)
操作: 返回首页 (Primary色),系管理员 (Secondary色)

### 13.5 404页面 (Not Found)

图标: 🔍 (gray-400)
主标题: 页面不存在 (gray-900)
副标题: 您访问的页面可能已被删除或移动 (gray-500)
操作: 返回首页 (Primary色), 查看文档 (Secondary色)

### 13.6 边界状态设计规范

布局:
- 居中对齐
- 图标: 48px
- 主标题: 18px, margin-top: 24px
- 副标题: 14px, margin-top: 8px
- 按钮: margin-top: 24px, 水平排列

颜色规范:
- 错误状态: red-500/600
- 警告状态: orange-500/600
- 信息状态: blue-500/600
- 成功状态: green-500/600
