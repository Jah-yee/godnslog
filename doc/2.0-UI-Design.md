## 一、设计目标

### 1.1 核心目标

将GODNSLOG从"Demo级工具"升级为"企业级OAST平台"，满足专业安全团队的高频使用需求。

### 1.2 设计原则

1. **层级清晰**: 侧边栏导航 + 面包屑，功能分组明确
2. **信息密度可控**: 列表/看板/时间线多视图切换
3. **详情独立页面**: 复杂对象(Case/Payload/Interaction)有独立详情页，非弹窗
4. **专业数据表格**: 分页、排序、筛选、列定制、批量操作
5. **暗色模式**: 支持light/dark/system主题
6. **响应式**: 桌面/平板/手机自适应

### 4.2 布局组件规范

**AppShell布局参数**:
- 侧边栏宽度: `240px` (展开) / `64px` (收起)
- 顶部栏高度: `64px`
- 内容区: `max-width: 1440px`, 居中
- 断点:
  - `lg`: 桌面 (>1024px), 侧边栏默认展开
  - `md`: 平板 (768px-1024px), 侧边栏可收起
  - `sm`: 手机 (<768px), 侧边栏变为抽屉

## 十一、当前实现与设计目标差异分析

### 11.1 导航结构差异

| 维度 | 当前实现 | 设计目标 | 差距等级 |
|------|---------|---------|---------|
| **布局** | 顶部导航条，9个平级菜单 | 侧边栏分组导航，三级结构 | 🔴 致命 |
| **层级** | 无分组，所有功能平铺 | OAST/Monitor/Automate/System分组 | 🔴 致命 |
| **移动端** | 无适配，小屏幕不可用 | 响应式，移动端变为抽屉/底部导航 | 🔴 致命 |
| **扩展性** | 新增菜单项将撑爆导航 | 侧边栏可滚动，分组可折叠 | 🟠 严重 |

**具体问题截图**:
