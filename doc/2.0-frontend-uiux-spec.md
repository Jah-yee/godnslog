# GODNSLOG 2.0 前端 UI/UX 规范

> 文档版本: 2026-05-05-v1
> 基于决策项: Q7 已确认
> 技术栈: Next.js 14 (App Router) + TypeScript + TailwindCSS + shadcn/ui

---

## 一、设计目标

1. **生产级可用**：不是Demo，而是安全团队每天使用的工具。
2. **信息密度优先**：安全工具需要在一屏内展示大量信息，避免过度留白。
3. **操作路径最短**：3次点击内完成核心操作（创建Case → 生成Payload → 查看命中）。
4. **实时感知**：新Interaction到达时必须有视觉/声音提示（可选）。
5. **专业可信**：深色侧边栏 + 浅色内容区，类似 Datadog / Splunk / Burp 的专业工具风格。

## 四、状态管理策略

### 4.1 服务端状态（Server State）→ TanStack Query

所有API数据使用TanStack Query管理：

```typescript
// features/cases/hooks/use-cases.ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { casesApi } from '../api'

export function useCases(params?: CaseListParams) {
  return useQuery({
    queryKey: ['cases', params],
    queryFn: () => casesApi.list(params),
  })
}

export function useCreateCase() {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: casesApi.create,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['cases'] })
    },
  })
}
```

### 4.2 客户端状态（Client State）→ Zustand

少量全局客户端状态使用Zustand：

```typescript
// features/auth/store.ts
import { create } from 'zustand'

interface AuthState {
  user: User | null
  token: string | null
  isAuthenticated: boolean
  setAuth: (user: User, token: string) => void
  logout: () => void
}

export const useAuthStore = create<AuthState>((set) => ({
  user: null,
  token: null,
  isAuthenticated: false,
  setAuth: (user, token) => set({ user, token, isAuthenticated: true }),
  logout: () => {
    localStorage.removeItem('token')
    set({ user: null, token: null, isAuthenticated: false })
  },
}))
```

**Zustand使用范围**（禁止滥用）：
- ✅ 认证状态（user/token）
- ✅ 主题/暗黑模式
- ✅ 侧边栏折叠状态
- ✅ 全局通知/Toast队列
- ❌ 表单数据（用RHF）
- ❌ 列表数据（用TanStack Query）
- ❌ 筛选条件（用URL query params）

## 六、表单设计规范

### 6.1 统一表单模式

所有表单使用 **React Hook Form + Zod + shadcn Form组件**：

```typescript
// 示例：Case创建表单
const formSchema = z.object({
  title: z.string().min(1, '标题不能为空').max(255),
  description: z.string().max(1000).optional(),
  target: z.string().max(255).optional(),
  tags: z.array(z.string()).max(10).optional(),
})

export function CaseForm({ onSubmit, defaultValues }: CaseFormProps) {
  const form = useForm<z.infer<typeof formSchema>>({
    resolver: zodResolver(formSchema),
    defaultValues,
  })

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(onSubmit)} className="space-y-4">
        <FormField
          control={form.control}
          name="title"
          render={({ field }) => (
            <FormItem>
              <FormLabel>标题</FormLabel>
              <FormControl>
                <Input placeholder="输入Case标题" {...field} />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        {/* ... */}
        <Button type="submit" disabled={form.formState.isSubmitting}>
          {form.formState.isSubmitting ? '提交中...' : '创建'}
        </Button>
      </form>
    </Form>
  )
}
```

### 6.2 表单校验规则

| 字段类型 | 规则 |
|---------|------|
| 标题/名称 | `min(1)`, `max(255)` |
| 描述 | `max(1000)` |
| 域名 | 可选校验正则 |
| Token | `readonly`，后端生成 |
| 日期 | `date` picker组件 |
| 数组（tags） | `max(10)` |

## 七、交互设计规范

### 7.1 数据加载状态

| 场景 | 展示方式 |
|------|---------|
| 初始加载 | Skeleton占位（shadcn Skeleton） |
| 刷新/重新加载 | Table行内Spinner或Button loading状态 |

### 7.2 操作反馈

| 操作 | 反馈方式 |
|------|---------|
| 创建成功 | Toast.success + 页面跳转/列表刷新 |
| 更新成功 | Toast.success + 保持当前页 |
| 删除成功 | Toast.success + 列表移除该行 |
| 操作失败 | Toast.error + 错误详情 |
| 复制成功 | Toast.success("已复制到剪贴板") |
| 无权限 | Toast.error + 引导联系管理员 |

### 7.3 确认Dialog规范

以下操作必须二次确认：
- 删除Case/Payload
- 归档Case
- 撤销APIKey
- 删除DNS解析记录
- 修改Rebinding配置

    </AlertDialogHeader>
    <AlertDialogFooter>
      <AlertDialogCancel>取消</AlertDialogCancel>
      <AlertDialogAction onClick={handleDelete} className="bg-red-500 hover:bg-red-600">确认删除</AlertDialogAction>

## 十、API集成规范

### 10.1 Axios实例配置

```typescript
// lib/api.ts
import axios from 'axios'

export const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || '/api/v2',
  timeout: 10000,
})

api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

api.interceptors.response.use(
  (response) => response.data,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)
```

### 10.4 类型契约同步（核心原则）

**前端类型必须来源于后端 Go struct，禁止独立编造。**

后端所有 API 请求/响应结构体声明在**独立可导出的包**（如 `pkg/apimodels/`），通过 `json:"field,omitempty"` tag 精确描述字段和类型。前端类型定义（`types/index.ts`）必须与这些 Go struct 保持严格同步。

**同步方式**（推荐优先级）：
1. **代码生成**（最佳）：使用 `quicktype` 等工具从 Go struct 自动生成 TypeScript interface，纳入 CI 检查。
2. **手工同步**（MVP可行）：每次修改后端 struct 后，手动更新前端对应类型，并在代码审查中强制检查。

**禁止行为**：
- ❌ 前端独立定义 `interface Case { ... }` 而不对照后端 struct
- ❌ 前端字段名/类型与后端不一致（如前端用 `camelCase` 但后端返回 `snake_case` 未通过 json tag 统一）
- ❌ 前端使用 `any` 绕过类型检查

**示例对照**：
```go
// backend: pkg/apimodels/case.go
type Case struct {
    ID          string    `json:"id"`
    Title       string    `json:"title"`
    Status      string    `json:"status"`
    Tags        []string  `json:"tags"`
    CreatedBy   string    `json:"created_by"`
    CreatedAt   time.Time `json:"created_at"`
}
```

```typescript
// frontend: types/index.ts
export interface Case {
  id: string
  title: string
  status: 'active' | 'archived' | 'completed'
  tags: string[]
  created_by: string
  created_at: string
}
```
