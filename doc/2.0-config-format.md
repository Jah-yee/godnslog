# GODNSLOG 2.0 Configuration Format

本文档定义GODNSLOG 2.0的配置格式和数据迁移边界。

## 配置文件格式

### 主配置文件: config.yaml

```yaml
# GODNSLOG 2.0 Configuration
version: "2.0"

# Server Configuration
server:
  # HTTP server
  http:
    addr: ":8080"
    read_timeout: 30s
    write_timeout: 30s
    idle_timeout: 120s
  
  # DNS server
  dns:
    addr: ":53"
    read_timeout: 5s
    write_timeout: 5s
    tcp_enabled: true
    udp_enabled: true
  
  # Domain configuration
  domain: "example.com"
  
  # IP addresses
  ipv4: "127.0.0.1"
  ipv6: "::1"  # optional

# Database Configuration
database:
  # Database type: sqlite, mysql, postgresql
  type: "sqlite"
  
  # SQLite specific
  sqlite:
    path: "./godnslog.db"
  
  # MySQL specific (when type=mysql)
  mysql:
    host: "localhost"
    port: 3306
    database: "godnslog"
    username: "godnslog"
    password: ""
    max_open_conns: 25
    max_idle_conns: 5
    conn_max_lifetime: 5m
  
  # PostgreSQL specific (when type=postgresql)
  postgresql:
    host: "localhost"
    port: 5432
    database: "godnslog"
    username: "godnslog"
    password: ""
    sslmode: "disable"
    max_open_conns: 25
    max_idle_conns: 5
    conn_max_lifetime: 5m

# Cache Configuration
cache:
  # Cache type: memory, redis
  type: "memory"
  
  # Memory cache specific
  memory:
    max_items: 10000
    cleanup_interval: 10m
  
  # Redis cache specific (when type=redis)
  redis:
    addr: "localhost:6379"
    password: ""
    db: 0
    pool_size: 10

# Authentication Configuration
auth:
  # JWT secret key (auto-generated if empty)
  jwt_secret: ""
  
  # Token expiration
  token_expiration: 24h
  
  # Session expiration in cache
  session_expiration: 24h
  
  # Password policy
  password_policy:
    min_length: 8
    require_uppercase: true
    require_lowercase: true
    require_digit: true
    require_special: false
  
  # Default admin user (created on first run)
  admin:
    username: "admin"
    email: "admin@example.com"
    password: ""  # auto-generated if empty

# APIKey Configuration
api_key:
  # Default expiration for new APIKeys
  default_expiration: 90d
  
  # Maximum expiration
  max_expiration: 365d
  
  # Supported scopes
  scopes:
    - "case:read"
    - "case:write"
    - "payload:read"
    - "payload:write"
    - "interaction:read"
    - "interaction:write"
    - "evidence:read"
    - "evidence:write"
    - "admin:all"

# Listener Configuration
listeners:
  # DNS listener
  dns:
    enabled: true
    upstream_dns: "8.8.8.8:53"
    ttl:
      default: 300
      log: 0
      ns: 600
      xip: 604800
  
  # HTTP listener
  http:
    enabled: true
    max_body_size: 10MB
  
  # SMTP listener (Phase 6)
  smtp:
    enabled: false
    addr: ":25"
  
  # LDAP listener (Phase 6)
  ldap:
    enabled: false
    addr: ":389"
  
  # SMB listener (Phase 6)
  smb:
    enabled: false
  
  # FTP listener (Phase 6)
  ftp:
    enabled: false

# Workflow Configuration
workflow:
  # Enable workflow engine
  enabled: true
  
  # Async queue
  queue:
    type: "memory"  # memory, redis
    max_workers: 5
  
  # Retry configuration
  retry:
    max_attempts: 3
    backoff: "exponential"
    initial_delay: 1s

# Notification Configuration
notification:
  # Enable notifications
  enabled: true
  
  # Webhook
  webhook:
    enabled: true
    timeout: 10s
    max_retries: 3
  
  # Email
  email:
    enabled: false
    smtp_host: ""
    smtp_port: 587
    smtp_user: ""
    smtp_password: ""
    from: ""
  
  # Feishu (Lark)
  feishu:
    enabled: false
    webhook_url: ""
  
  # WeCom (WeChat Work)
  wecom:
    enabled: false
    webhook_url: ""
  
  # DingTalk
  dingtalk:
    enabled: false
    webhook_url: ""
  
  # Slack
  slack:
    enabled: false
    webhook_url: ""
  
  # Discord
  discord:
    enabled: false
    webhook_url: ""

# Logging Configuration
logging:
  # Log level: debug, info, warn, error
  level: "info"
  
  # Log format: json, text
  format: "json"
  
  # Log output: stdout, file
  output: "stdout"
  
  # File output (when output=file)
  file:
    path: "./logs/godnslog.log"
    max_size: 100MB
    max_age: 7d
    max_backups: 10

# Audit Log Configuration
audit:
  # Enable audit logging
  enabled: true
  
  # Retention period
  retention: 90d
  
  # Log level: all, important, critical
  level: "all"

# Frontend Configuration
frontend:
  # Enable frontend
  enabled: true
  
  # Serve static files
  static_dir: "./frontend-next/dist"
  
  # Enable guest user
  guest_enabled: false
  
  # Default language
  default_language: "en-US"

# Migration Configuration
migration:
  # Enable 1.0 to 2.0 migration
  auto_migrate: true
  
  # Backup before migration
  backup_before_migration: true
  
  # Backup path
  backup_path: "./backups"

# Feature Flags
features:
  # Canary tokens (Phase 6)
  canary_enabled: false
  
  # Rebinding Lab (Phase 6)
  rebinding_lab_enabled: false
  
  # DNS C2 (Phase 6, experimental)
  dns_c2_enabled: false
  
  # AI summarization (Phase 6, optional)
  ai_summarization_enabled: false
```

## 环境变量支持

所有配置项都支持通过环境变量覆盖，使用 `GODNSLOG_` 前缀和下划线分隔：

```bash
# 示例
export GODNSLOG_SERVER_HTTP_ADDR=":9090"
export GODNSLOG_DATABASE_TYPE="mysql"
export GODNSLOG_DATABASE_MYSQL_HOST="db.example.com"
export GODNSLOG_AUTH_JWT_SECRET="your-secret-key"
```

## 数据迁移边界

### 1.0到2.0迁移策略

### 数据模型映射

#### 用户表迁移
```go
// 1.0 TblUser -> 2.0 User + UserProfile
TblUser {
    Id -> User.ID
    Name -> User.Username
    Email -> User.Email
    Pass -> User.PasswordHash
    Role -> User.Role
    ShortId -> User.ShortID
    Token -> User.APIToken (deprecated)
    Lang -> UserProfile.Language
    Callback -> UserProfile.CallbackURL
    CallbackMessage -> UserProfile.CallbackTemplate
    Rebind -> UserProfile.RebindingIPs
    CleanInterval -> UserProfile.CleanInterval
}
```

#### DNS记录迁移
```go
// 1.0 TblDns -> 2.0 Interaction (type=dns)
TblDns {
    Id -> Interaction.ID
    Uid -> Interaction.UserID
    Domain -> Interaction.Domain
    Var -> Interaction.Token (部分)
    Ip -> Interaction.SourceIP
    Ctime -> Interaction.Timestamp
    // 新增字段
    -> Interaction.Type = "dns"
    -> Interaction.CaseID (NULL)
    -> Interaction.PayloadID (NULL)
}
```

#### HTTP记录迁移
```go
// 1.0 TblHttp -> 2.0 Interaction (type=http)
TblHttp {
    Id -> Interaction.ID
    Uid -> Interaction.UserID
    Ip -> Interaction.SourceIP
    Var -> Interaction.Token (部分)
    Path -> Interaction.Path
    Method -> Interaction.Method
    Data -> Interaction.Body
    Ctype -> Interaction.ContentType
    Ua -> Interaction.UserAgent
    Ctime -> Interaction.Timestamp
    // 新增字段
    -> Interaction.Type = "http"
    -> Interaction.CaseID (NULL)
    -> Interaction.PayloadID (NULL)
}
```

#### DNS解析记录保留
```go
// 1.0 TblResolve -> 2.0 ResolveRecord (保持不变)
TblResolve {
    // 直接复制，结构保持兼容
}
```

### 迁移步骤

#### Phase 0: 准备阶段
1. 备份现有数据库
2. 检查配置文件兼容性
3. 创建新表结构
4. 运行迁移脚本

#### Phase 1: 数据迁移
1. 迁移用户数据
2. 迁移DNS记录到Interaction表
3. 迁移HTTP记录到Interaction表
4. 保留TblResolve表
5. 创建索引

#### Phase 2: 验证阶段
1. 验证用户登录
2. 验证DNS记录查询
3. 验证HTTP记录查询
4. 验证权限系统
5. 性能测试

#### Phase 3: 清理阶段
1. 重命名旧表为TblDns_v1, TblHttp_v1
2. 保留一段时间供回滚
3. 确认无问题后删除旧表

### 配置迁移

#### 1.0命令行参数 -> 2.0配置文件
```bash
# 1.0
godnslog serve -domain example.com -4 127.0.0.1

# 2.0 (config.yaml)
server:
  domain: "example.com"
  ipv4: "127.0.0.1"
```

### API兼容性

#### 保留的1.0 API端点
- POST /user/login
- POST /user/logout
- GET /dns/record
- GET /http/record
- GET /api/dns (Token认证)
- GET /api/http (Token认证)

#### 新增的2.0 API端点
- /api/v2/cases/*
- /api/v2/payloads/*
- /api/v2/interactions/*
- /api/v2/evidence/*
- /api/v2/apikeys/*
- /api/v2/audit/*

#### API版本策略
- 1.0 API保持兼容，标记为deprecated
- 2.0 API使用/api/v2前缀
- 逐步引导用户迁移到新API

### 回滚策略

#### 回滚条件
- 数据迁移失败
- 性能严重下降
- 关键功能不可用
- 用户报告严重问题

#### 回滚步骤
1. 停止2.0服务
2. 恢复数据库备份
3. 恢复1.0配置
4. 启动1.0服务
5. 分析失败原因

### 数据保留策略

#### 保留期限
- Interaction记录: 默认90天，可配置
- Audit日志: 默认90天，可配置
- Case: 永久保留
- Payload: 永久保留
- Evidence: 永久保留

#### 清理策略
- 基于用户配置的CleanInterval
- 支持手动清理
- 支持按时间范围清理
- 支持按Case清理

## 迁移脚本

### 自动迁移脚本
提供自动迁移工具：
```bash
godnslog migrate --from 1.0 --to 2.0 --backup
```

### 手动迁移步骤
如需手动控制迁移过程：
1. 备份数据库
2. 运行SQL迁移脚本
3. 验证数据完整性
4. 更新配置文件
5. 重启服务

## 验证清单

迁移完成后需验证：
- [ ] 所有用户可以登录
- [ ] DNS记录正常显示
- [ ] HTTP记录正常显示
- [ ] Rebinding功能正常
- [ ] 标准DNS解析正常
- [ ] 权限系统正常
- [ ] 性能无明显下降
- [ ] 日志正常输出
- [ ] 审计日志正常记录
